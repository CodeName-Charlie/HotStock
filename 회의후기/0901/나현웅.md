```
1) Q : 크롤링으로 기사를 가져오는 대신 네이버 api를 사용하여 데이터를 가져오는 것으로 변경?
    >>> A : 잘 가져온다면 크롤링보다 api가 낫지만 제한적일 수 있음
2) Q : 가져오는 기사의 범위와 예상되는 소요 판단(기사 전문을 가져와야하는가) ?
    >>> A : DB에 전문을 저장하고 보여주는 것은 저작권 쪽 문제가 발생할 수 있음.
3) Q : 키워드 리스트는 메모리에서? DB에서? 어느 쪽에서 할 것인가 ?
    >>> A : 메모리에서 하는 것은 위험함. DB 혹은 인 메모리 DB(Redis) 권장
4) Q : 기간 지정 키워드 추출 (해당 기간 키워드 구간합) 구조를 고려한 세그먼트트리 DB가 필요한가?
    >>> A : 가능은 할 수 있지만 쓰는 거 본 적 없음.
5) Q : 유사 키워드( ex. 음료, 음료수 ) 가 나뉘어 저장되는 경우의 처리 관련
    >>> A : 소요가 큰 것은 사실. 본인이라면 그래도 한 번 더 처리 할 것이라고 말씀하심.
6) Q : UI는 어떻게 할 것인가? (보여주어야 할 정보가 많음)
    >>> A : 서버 사이드 렌더링을 포함하여 프론트 쪽 소요가 클 것으로 예상됨.
    >>>     서로 진행 상황을 잘 공유할 것 권장
```
